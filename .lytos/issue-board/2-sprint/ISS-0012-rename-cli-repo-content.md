# ISS-0012 ‚Äî Replace all socle references in CLI repo (lytos-cli)

## Context

The CLI repo contains ~240 occurrences across ~42 files. This includes the npm package name, CLI binary name, hardcoded URLs, `.socle/` directory paths, and brand references in templates and tests.

## Proposed solution

Systematic rename in order: package identity first, then source code, then docs.

## Checklist

### Package identity
- [ ] `package.json` name: `le-socle` ‚Üí `lytos`
- [ ] `package.json` bin: `"socle"` ‚Üí `"lytos"`
- [ ] `package.json` repository URL ‚Üí `github.com/getlytos/lytos-cli`
- [ ] `package.json` homepage ‚Üí `github.com/getlytos/lytos-method`
- [ ] `package.json` bugs URL ‚Üí `github.com/getlytos/lytos-cli/issues`
- [ ] `package.json` keywords: update
- [ ] `package-lock.json` ‚Äî regenerate after changes

### CLI entry point
- [ ] `src/cli.ts` ‚Äî `.name("socle")` ‚Üí `.name("lytos")`
- [ ] `src/cli.ts` ‚Äî description text

### Source code URLs
- [ ] `src/lib/templates.ts` ‚Äî `REPO_URL` constant ‚Üí `github.com/getlytos/lytos-method`
- [ ] `src/lib/scaffold.ts` ‚Äî `REPO_RAW` constant ‚Üí `raw.githubusercontent.com/getlytos/lytos-method`
- [ ] `src/commands/init.ts` ‚Äî documentation link

### Directory references (`.socle/` ‚Üí `.lytos/`)
- [ ] `src/commands/init.ts`
- [ ] `src/commands/board.ts`
- [ ] `src/lib/scaffold.ts`
- [ ] `src/lib/templates.ts`
- [ ] All files in `src/`

### File references (`SOCLE.md` ‚Üí `LYTOS.md`)
- [ ] `src/lib/templates.ts`
- [ ] `src/lib/scaffold.ts`

### Tests

- [ ] `tests/commands/board.test.ts` ‚Äî directory paths
- [ ] `tests/helpers/fixtures.ts` ‚Äî directory paths

### Documentation & config
- [ ] `README.md` ‚Äî badges, install commands, brand name

- [ ] `.socle/` directory ‚Üí `.lytos/` (rename + update contents)

### Brand name in user-facing messages
- [ ] All `console.log` / user messages in `src/commands/`

## Relevant files

~42 files. Key files:
- `package.json`

- `README.md`, `CLAUDE.md`

- `grep -ri "socle" . --exclude-dir=node_modules --exclude-dir=.git` returns zero matches
- `npm run build` succeeds
